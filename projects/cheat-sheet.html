checkout, branch

checkout a branch: 


git checkout name-of-branch

checkout a branch and create it at the same time:


git checkout -b name-of-branch
restore working directory to last commit: 

git checkout HEAD

Update all files in working directory to match the specified commit: 


git checkout specified-commit

access an older version of a single file:


git checkout specified-commit filename

display working branch:


git branch

create new branch


git branch branch-name

delete branch


git branch -d branch-name

add, commit, diff

add a file to staging area:


git add file-name

add everything in directory:


git add .

Begin an interactive staging session that lets you choose portions of a file to add to the next commit. This will present you with a chunk of changes and prompt you for a command. Use y to stage the chunk, n to ignore the chunk, s to split it into smaller chunks, e to manually edit the chunk, and q to exit.


git add -p

permanently store changes from the staging area inside the repo with message


git commit -m "Your message"

show the differences between the version you're working on and the one in the staging area:(press q to exit diff mode)


git diff file-name

clone, init

retrieve a complete replica of a remote repo:


git clone remote-location clone-name

transform the current directory into a git repo


git init

create a new folder containing only the git subdirectory:


git init directory-name

initialize a git repo but ommit the working directory:


git init --bare directory-name

status, log

show status of working directory and staging area:


git status

show a list of all previous commits.SHAs are the 40 character codes that serve as IDs for each commit.


git log

condesnse each commit to a single line:


git log --oneline

Along with normal info, show which files were altered and relative number of lines added or deleted:


git log --stat

searches for commits with a commit message that matches pattern which can be a string or regex:


git log --grep="pattern"

only display commits that include specified file: 


git log file-name

HEAD

In Git, the SHA of the commit you are currently on is known as the HEAD commit. In many cases, the most recently made commit is the HEAD commit. HEAD is the commit you are currently on, HEAD^ is the last commit, HEAD~3, HEAD~4, HEAD~5, are the 3rd 4th and 5th commit from HEAD respectively. To see the HEAD commit, enter:


git show HEAD

discard changes made for a file and return to HEAD commit:


git checkout HEAD file-name

Unstage file changes in the staging area:


git reset HEAD

reset

rewind to a previous commit using the first 7 characters of the SHA of the commit or using HEAD:


git reset commit-SHA
git reset HEAD

remove file from staging area but leave working directory unchanged:


git reset file-name

remote, fetch, push, merge

A remote is a shared Git repository that allows multiple collaborators to work on the same Git project from different locations. Collaborators work on the project independently, and merge changes together when they are ready to do so.

List the remote repos:


git remote -v

push your branch to the remote origin:


git push origin your-branch-name

Fetch a branch from the remote into the local copy:


git fetch remote branch-name

git merge branch-name


git merge branch-name

fetch the remote's copy and immediately merge it:


git pull remote branch-name

summarized basic workflow

Here's what a workday is generally like:


git checkout -b branch-name

do some work on some files


git add whatever-needs-to-be-added
git commit -m "Message about what was done"
git push origin branch-name

In github, merge in branch. Delete branch, once successful.


git checkout master
git fetch origin master
git merge origin/master
git branch -d branch-name
